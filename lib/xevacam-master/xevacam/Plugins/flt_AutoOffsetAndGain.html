Image filter documentation: Auto offset and gain

AutoOffsetAndGain Programming name
Programming name to be used in Xeneth GUI application settings "Auto start filters" or when queueing the filter while programming with Xeneth SDK.
Not available on Linux.
OffsetControl Offset control [Read/Write]
In manual mode the Offset is applied to the image. Otherwise the offset (see Offset used) is computed based on the image content.
  Values: Automatic(0), Manual(1)
GainControl Gain control [Read/Write]
In manual mode the Gain is applied to the image. Otherwise the gain (see Gain used) is computed based on the image content.
  Values: Automatic(0), Manual(1)
AutoModeUpdate Auto mode update [Read/Write]
When stopped, the computed values for offset and gain are frozen.
  Values: Running(0), Stopped(1)
OffsetUsed Offset used (%) [Read]
Offset currently used.
  Values: number in [-100,100]
GainUsed Gain used [Read]
Gain currently used.
  Values: positive number
ManualModeOffset Offset (%) [Read/Write]
This value is applied when OffsetControl is manual.
  Values: number in [-100,100]
ManualModeGain Gain [Read/Write]
This value is applied when GainControl is manual.
  Values: positive number
AutoModeOutputHistogramMidpoint Output histogram midpoint (%) [Write/Read]
The target value for the histogram midpoint when OffsetControl is Automatic.
More precisely the value for Input histogram midpoint is mapped onto the output histogram midpoint.
  Values: number in [0,100]
AutoModeMaximalGain Maximal gain [Read/Write]
The maximal gain applied when GainControl is Automatic.
  Values: positive number.
AutoModeRegulationSpeed Regulation speed [Read/Write]
Defines the response time to changes in the image.
When zero, the response immediate.
  Values: integer in [0,9]
OutliersExpressedAs Outliers expressed as [Read/Write]
Outliers are pixels at the start or at the end of the image histogram that are not taken into account when Offset used and Gain used are computed.
When put to "% of current image size", a fixed amount of pixels is ignored. Otherwise pixels are ignored when their value is too low/too high.
The thresholds are define by Dark/Bright.
  Values: % of current image size (0),% of maximal intensity (1)
OutliersDark Dark (%) [Read/Write]
Threshold for the dark outliers. See Outliers expressed as.
  Values: number in [0,100]
OutliersBright Bright (%) [Read/Write]
Threshold for the bright outliers. See Outliers expressed as.
  Values: Values: number in [0,100]
OutliersBrightDisplayColor Bright display colour [Read/Write]
To stress that bright outliers do not contain valuable information, they can be coloured black.
  Values: White(0), Black(1)
InputHistogramBegin Input histogram begin (%) [Read/Write]
The relevant input for the algorithm is the image histogram minus the outliers.
When setting the Input histogram begin to a strictly positive value the start of the histogram taken into account is shifted up by ignoring the darker part of the image.
The value is expressed as a percentage of the relevant histogram width.
  Values: number in [0,100]
InputHistogramMidpoint Input histogram midpoint (%) [Read/Write]
The relevant input for the algorithm is the image histogram minus the outliers.
The input histogram midpoint allows redefing the point in the input histogram that ends up at the Output histogram midpoint.
The value is expressed as a percentage of the relevant histogram width.
  Values: number in [0,100]
InputHistogramEnd Input histogram end (%) [Read/Write]
The relevant input for the algorithm is the image histogram minus the outliers.
When setting the Input histogram end to a value strictly less than 100% the end of the histogram taken into account is shifted down by ignoring the brighter part of the image.
The value is expressed as a percentage of the relevant histogram width.
  Values: number in [0,100]
WindowOffsetX Offset X [Read/Write]
Only pixels in the window defined by Offset X, Offset Y, Width and Height are taken into account.
  Values: integer in [0,width of the image - 1]
WindowOffsetY Offset Y [Read/Write]
Only pixels in the window defined by Offset X, Offset Y, Width and Height are taken into account.
  Values: integer in [0,height of the image - 1]
WindowWidth Width [Read/Write]
Only pixels in the window defined by Offset X, Offset Y, Width and Height are taken into account.
  Values: integer in [1,width of the image]
WindowHeight Height [Read/Write]
Only pixels in the window defined by Offset X, Offset Y, Width and Height are taken into account.
  Values: integer in [1,height of the image]
