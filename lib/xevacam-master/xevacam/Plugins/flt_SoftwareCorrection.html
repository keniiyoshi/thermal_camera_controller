Image filter documentation: Software correction

SoftwareCorrection Programming name
Programming name to be used when queueing the filter while programming with Xeneth SDK.
Mode Mode [Read/Write]
Correct: bad pixels are replaced and a non uniformity correction is applied.
Show offset: the data used to compensate for fixed pattern noise (FPN) is shown.
Show gain: the data used to compensate for photon response non uniformity (PRNU) is shown.
  Values: Correct(0), Show offset(1), Show gain(2), Passthrough(2)
Replacement Replacement [Read/Write]
Select the algorithm to be applied for replacing bad pixels.

In case of a 2D sensor the following strategies are available:
None: no pixels are replaced.
Adaptive: a representative neighbour is copied.
Vertical: the pixel above is copied.
Vertical interpolation: interpolation between two pixels in the same column.
Horizontal: the left pixel copied.
Horizontal interpolation: interpolation between two pixels in the same row.

In case of a line array the following strategies are available:
None: no pixels are replaced.
Linear: the left pixel is copied.

  Values(2D): None(0), Adaptive(1), Vertical(2), Vertical interpolation(3), Horizontal(4), Horizontal interpolation(5)
  Values(1D): None(0), Linear(1)
KeepOffset Keep offset [Read/Write]
No: the FPN correction (dark current) is subtracted from the signal.
Yes: the overall image level is preserved.
  Values: Yes(0), No(1)
DarkOffset Offset correction [Read/Write]
Global offset that is subtracted from the FPN correction data before the correction is applied.
  Values: integer
BlackClip Black clip [Read/Write]
Pixel values below this threshold are clipped to the value.
  Values: integer
WhiteClip White clip [Read/Write]
Pixel values above this threshold are clipped to the value.
  Values: integer
GainAdjustment Gain multiplier [Read/Write]
Factor multiplied to the response of the camera.
  Values: floating number
